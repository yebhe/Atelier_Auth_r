Bienvenue dans votre atelier 1

Votre objectif est de sécuriser l'accès à cette page via une solution d'authentification basique (Basic Auth)

Pour sécuriser une page web avec un système d'authentification basique, vous devez configurer votre serveur web pour exiger la saisie d'un nom d'utilisateur et d'un mot de passe

Tout ce dont vous avez besion de savoir se trouve ici : Lien d'aide

- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

Ce que nous dit l'aide en ligne d'Alwaysdata :

Etape 1 : Il faut créer dans le répertoire à protéger un fichier .htaccess contenant le code suivant 


AuthName "Accès protégé"
AuthType Basic
AuthUserFile /home/{votre_compte_Alwaysdata}/www/.htpasswd
Require valid-user
Attention : la chaine {votre_compte_Alwaysdata} est à remplacer par votre compte Alwaysdata.

Etape 2 : Création d'un fichier .htpasswd à la racine de notre serveur (c'est à dire, dans notre cas, à la racine de notre repository GitHub), contenant les couples d'identifiants et mots de passes autorisés à ce connecter à notre page.

Pour générer le couple login/password contenu dans le fichier .htpasswd vous pouvez utiliser la solution en ligne suivante : Générateur de fichier .htpasswd
